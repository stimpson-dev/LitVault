<task type="auto">
  <name>Task 1: Implement IngestService pipeline</name>
  <files>backend/app/ingest/service.py</files>
  <action>
  Create `backend/app/ingest/service.py` with:

  1. `@dataclass` `IngestResult`:
     - total_found: int
     - new_files: int
     - processed: int
     - errors: int
     - skipped: int

  2. `class IngestService`:
     - `__init__(self, db: AsyncSession, settings: Settings)`

     - `async def ingest_folder(self, folder: str) -> IngestResult` — Main pipeline:
       a. Validate folder exists and is accessible (Path.is_dir, with timeout for network)
       b. Call `find_new_files(folder, self.db)` to get list of new/changed file metas
       c. For each file meta:
          - Create or update Document row with processing_status="processing", file metadata fields (file_path, file_hash, file_type, file_size, mtime)
          - Call `parse_document(Path(meta["file_path"]), meta["file_type"])`
          - If ParseResult.error: set processing_status="error", store error in a log field or summary
          - If success: set full_text=result.text, has_text=result.has_text, processing_status="done", indexed_at=now
          - If PDF: call generate_thumbnail_async, store thumbnail path if generated
          - Commit after each document (not batch — single writer safety)
       d. Return IngestResult with counts

     - `async def _upsert_document(self, meta: dict) -> Document` —
       - Check if document with same file_path exists → update
       - Otherwise create new Document
       - Return the Document object

     Use `from sqlalchemy import select` for queries.
     Use `logger = logging.getLogger("litvault.ingest")`

     IMPORTANT: After inserting/updating a Document with full_text, the FTS5 triggers
     will auto-sync the search index. No manual FTS5 insert needed.

     For thumbnail output_dir: use `Path(settings.thumbnails_dir)` (from config).
  </action>
  <verify>
  `cd backend && uv run python -c "from app.ingest.service import IngestService, IngestResult; print('OK')"`
  </verify>
  <done>
  IngestService.ingest_folder: scans folder, parses new/changed files, stores in DB with processing status. Thumbnails generated for PDFs. Errors logged, not raised. FTS5 auto-synced via triggers.
  </done>
</task>
