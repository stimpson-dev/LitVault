  <name>Task 1: Implement folder crawler with incremental detection</name>
  <files>backend/app/ingest/__init__.py, backend/app/ingest/crawler.py</files>
  <action>
  Create `backend/app/ingest/__init__.py` (empty).

  Create `backend/app/ingest/crawler.py` with:

  1. `SUPPORTED_EXTENSIONS = {".pdf", ".docx", ".pptx"}`

  2. `hash_file(path: Path) -> str` — SHA-256 hash of file contents. Read in 8KB chunks. Wrap in asyncio.to_thread with 30s timeout (asyncio.wait_for). Return hex digest.

  3. `collect_file_meta(path: Path) -> dict` — Returns dict with:
     - `file_path`: str (absolute, forward slashes)
     - `file_type`: str (extension without dot, lowercase)
     - `file_size`: int (bytes)
     - `mtime`: float (os.path.getmtime)
     - `file_hash`: str (from hash_file)

  4. `scan_folder(folder: str | Path) -> AsyncGenerator[dict, None]` —
     - Use asyncio.to_thread wrapping os.walk (network drives can hang)
     - asyncio.wait_for with 60s timeout per folder level
     - Filter by SUPPORTED_EXTENSIONS
     - yield collect_file_meta for each file
     - Log and skip files that fail (permission errors, timeouts)

  5. `find_new_files(folder: str | Path, db: AsyncSession) -> list[dict]` —
     - Call scan_folder to get all file metas
     - Query DB for existing documents: SELECT file_path, file_hash FROM documents
     - Compare: file is new if path not in DB, or hash changed
     - Return list of file metas for new/changed files only

  Use `import logging; logger = logging.getLogger("litvault.crawler")`
  All file I/O wrapped in try/except with logging (never crash on single file).
  Use Path objects internally, store as forward-slash strings in metadata.
  </action>
  <verify>
  `cd backend && uv run python -c "from app.ingest.crawler import scan_folder, find_new_files, hash_file; print('OK')"`
  </verify>
  <done>
  crawler.py imports cleanly. scan_folder yields file metadata dicts for supported types. find_new_files compares against DB. hash_file produces SHA-256 hex digest. All I/O has timeout protection.
  </done>
</task>
