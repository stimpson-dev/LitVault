  <name>Task 1: FTS5 query sanitizer</name>
  <files>backend/app/search/__init__.py, backend/app/search/sanitizer.py</files>
  <action>
  Create `backend/app/search/__init__.py` (empty).

  Create `backend/app/search/sanitizer.py` with:

  1. `def sanitize_fts5_query(query: str) -> str`:
     - FTS5 treats hyphens as NOT operator — this is the #1 pitfall!
     - Steps:
       a. Strip leading/trailing whitespace
       b. If empty: return ""
       c. Replace hyphens with spaces: `query.replace("-", " ")`
       d. Remove FTS5 special characters that could break queries: `*`, `"`, `(`, `)`, `:`, `^`
          Use re.sub to remove: `re.sub(r'[*"():^]', '', query)`
       e. Collapse multiple spaces to single: `re.sub(r'\s+', ' ', query).strip()`
       f. If result is empty after sanitization: return ""
       g. Wrap each word in double quotes for exact matching (prevents operator interpretation):
          `" ".join(f'"{word}"' for word in result.split())`
       h. Return the sanitized query

     This approach quotes each term individually, preventing any accidental FTS5 operators.

  2. `def sanitize_fts5_query_with_prefix(query: str) -> str`:
     - Same as above but append `*` to the last term for prefix matching (autocomplete-style)
     - Useful for search-as-you-type

  Use `import re`
  </action>
  <verify>
  ```bash
  cd backend && uv run python -c "
from app.search.sanitizer import sanitize_fts5_query, sanitize_fts5_query_with_prefix

# Hyphens (the big pitfall)
assert 'NOT' not in sanitize_fts5_query('Zahnrad-Geometrie')
result = sanitize_fts5_query('Zahnrad-Geometrie')
assert '\"Zahnrad\"' in result and '\"Geometrie\"' in result

# Special chars removed
result = sanitize_fts5_query('FEM (Finite*Element)')
assert '(' not in result and '*' not in result

# Empty
assert sanitize_fts5_query('') == ''
assert sanitize_fts5_query('---') == ''

# Prefix matching
prefix = sanitize_fts5_query_with_prefix('Zahnr')
assert prefix.endswith('*')

print('OK')
"
  ```
  </verify>
  <done>
  FTS5 query sanitizer handles hyphens (→ spaces), special characters, and wraps terms in quotes. Prefix variant for search-as-you-type.
  </done>

<task type="auto">
  <name>Task 2: Search service with FTS5, filters, facets</name>
  <files>backend/app/search/service.py</files>
  <action>
  Create `backend/app/search/service.py` with:

  1. `@dataclass` `SearchFilters`:
     - category: str | None = None
     - doc_type: str | None = None
     - year_min: int | None = None
     - year_max: int | None = None
     - language: str | None = None
     - author: str | None = None
     - has_text: bool | None = None

  2. `@dataclass` `SearchResult`:
     - documents: list[dict]
     - total: int
     - facets: dict  # {"categories": [...], "doc_types": [...], "years": [...]}

  3. `class SearchService`:
     - `__init__(self, db: AsyncSession)`

     - `async def search(self, query: str, filters: SearchFilters | None = None, offset: int = 0, limit: int = 50) -> SearchResult`:
       a. Sanitize query via sanitize_fts5_query()
       b. If query is non-empty: use FTS5 search with BM25 ranking
          ```sql
          SELECT d.*,
                 snippet(documents_fts, 0, '<mark>', '</mark>', '...', 32) as title_snippet,
                 snippet(documents_fts, 2, '<mark>', '</mark>', '...', 64) as text_snippet,
                 bm25(documents_fts, 10.0, 5.0, 1.0, 2.0) as rank
          FROM documents_fts
          JOIN documents d ON d.id = documents_fts.rowid
          WHERE documents_fts MATCH :query
          ORDER BY rank
          ```
          Column weights in bm25(): title=10, authors=5, full_text=1, summary=2
       c. If query is empty: return all documents ordered by created_at DESC
       d. Apply metadata filters as WHERE clauses:
          - category filter: JOIN document_categories dc JOIN categories c WHERE c.name = :category
          - doc_type: WHERE d.doc_type = :doc_type
          - year_min/max: WHERE d.year >= :year_min AND d.year <= :year_max
          - language: WHERE d.language = :language
          - author: WHERE d.authors LIKE '%' || :author || '%'
          - has_text: WHERE d.has_text = :has_text
       e. Get total count (separate COUNT query with same filters but no LIMIT/OFFSET)
       f. Apply pagination: LIMIT :limit OFFSET :offset
       g. Get facet counts (parallel queries):
          - categories: COUNT grouped by category name (via join)
          - doc_types: COUNT grouped by doc_type
          - years: COUNT grouped by year
       h. Return SearchResult

     The SQL should be built using raw text() queries for the FTS5 parts
     (SQLAlchemy ORM doesn't natively support FTS5 MATCH).
     Use `from sqlalchemy import text` for raw SQL.
     Bind parameters properly (never interpolate user input!).

     - `async def get_facets(self, query: str = "", filters: SearchFilters | None = None) -> dict`:
       Helper that returns just the facet counts without document results.

  IMPORTANT security note: All user input MUST go through bind parameters (`:param`),
  never f-string interpolation. The sanitizer is an additional safety layer for FTS5
  syntax, but parameterized queries are the primary SQL injection defense.

  Use `import logging; from dataclasses import dataclass`
  Use `logger = logging.getLogger("litvault.search")`
  </action>
