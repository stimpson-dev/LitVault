       - category: str | None = None
       - doc_type: str | None = None

       - year_max: int | None = None

       - author: str | None = None
       - offset: int = 0
       - limit: int = 50
     - Create SearchFilters from query params
     - Create SearchService(db)
     - Call search(q, filters, offset, limit)
     - Return JSON: {"documents": [...], "total": N, "facets": {...}, "query": q, "filters": {...}}

  3. `GET /api/search/facets` endpoint:
     - Same filter params as search (minus offset/limit)
     - Return facet counts only (lighter endpoint for filter sidebar)

<task type="auto">
  <name>Task 2: Saved searches CRUD + duplicate check + wiring</name>
  <files>backend/app/search/router.py, backend/app/documents/router.py, backend/app/main.py</files>
  <action>
  Update `backend/app/search/router.py` — add saved searches endpoints:

  1. Pydantic models:
     - `SavedSearchCreate(BaseModel)`: name: str, query: str (JSON string of filter state)
     - `SavedSearchResponse(BaseModel)`: id: int, name: str, query: str, created_at: str

  2. `POST /api/saved-searches` — create saved search
     - Body: SavedSearchCreate
     - Create SavedSearch row
     - Return SavedSearchResponse

  3. `GET /api/saved-searches` — list all saved searches
     - Return list of SavedSearchResponse

  4. `GET /api/saved-searches/{search_id}` — get by id
     - 404 if not found

  5. `DELETE /api/saved-searches/{search_id}` — delete saved search
     - 404 if not found
     - Return {"deleted": true}

  Update `backend/app/documents/router.py` — add duplicate check:

  6. `GET /api/documents/duplicates` endpoint:
     - Query param: hash: str (SHA-256 hash)
     - Query documents WHERE file_hash = :hash
     - Return list of matching documents (id, file_path, file_hash, title)
     - This endpoint must be defined BEFORE the `/{doc_id}` route to avoid path conflict

  Update `backend/app/main.py`:
     - Import: `from app.search.router import router as search_router`
     - Add: `app.include_router(search_router)` after documents_router
  </action>
