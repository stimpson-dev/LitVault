# Plan 08-02: Metadaten aus Dateinamen extrahieren (ohne AI)

## Goal
Ohne AI-Classification trotzdem brauchbare Metadaten: Titel, Jahr, Dokumenttyp aus Dateinamen ableiten.

## Current Problem
Ohne Ollama bleiben title, year, doc_type, authors alle NULL. Der Dateiname enthält oft genug Information.

## Tasks

### T1: Filename-Metadata-Extractor erstellen
**File**: `backend/app/classification/filename_extractor.py` (NEU)
- Regex-basiert:
  - Jahr: 4-stellige Zahl (1900-2099)
  - Titel: Dateiname ohne Extension, bereinigt (Unterstriche → Leerzeichen, Datum-Prefix entfernt)
  - Dokumenttyp-Heuristik: "FVA-Heft" → "report", "Analyse" → "report", "Präsentation" → "presentation"
  - Sprache: Dateiendung-unabhängig, aus Dateiname-Heuristik
- Return: dict mit title, year, doc_type (alle optional)

### T2: IngestService — Filename-Metadaten als Fallback
**File**: `backend/app/ingest/service.py`
- Nach parse_document: wenn kein AI-Classifier, Filename-Extractor aufrufen
- Nur Felder setzen die noch NULL sind (überschreibt keine AI-Daten)
- `classification_source = "filename"`

## Verification
- [ ] "20170718 FEM Analyse HAG188.pdf" → title="FEM Analyse HAG188", year=2017, doc_type="report"
- [ ] "FVA-Heft 0032 - Wälzfestigkeit.pdf" → title="Wälzfestigkeit", year=None, doc_type="report"
- [ ] Bestehende AI-Metadaten werden nicht überschrieben
