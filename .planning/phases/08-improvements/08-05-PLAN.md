# Plan 08-05: Nachträglicher gezielter AI-Scan

## Goal
Einzelne Dokumente oder Gruppen können nachträglich per AI klassifiziert werden — manuell ausgelöst, unabhängig vom Crawl.

## Current State
- CLASSIFY Job-Typ existiert bereits im Worker (einzeln + batch)
- Ollama-Client vorhanden, aber aktuell wegen RAM-Problemen deaktiviert beim Crawl
- AI-Classification Code in IngestService._apply_classification() funktionsfähig

## Tasks

### T1: Backend — Classify-Endpoint verbessern
**File**: `backend/app/documents/router.py`
- `POST /api/documents/{id}/classify` → queued CLASSIFY-Job für einzelnes Dokument
- `POST /api/documents/classify-batch` → Body: `{ "ids": [1, 2, 3] }` oder `{ "filter": "unclassified" }`
- Prüft ob Ollama erreichbar ist, gibt 503 mit klarer Meldung wenn nicht
- Response: Job-ID für Progress-Tracking

### T2: Frontend — AI-Scan Button in DocumentDetail
**File**: `frontend/src/components/DocumentDetail.tsx`
- Button "KI-Analyse starten" bei Dokumenten ohne classification_source
- Button "KI-Analyse wiederholen" bei bereits klassifizierten
- Zeigt Spinner während Job läuft
- Nach Abschluss: Metadaten automatisch neu laden

### T3: Frontend — Batch-Classify in Toolbar
**File**: `frontend/src/components/Toolbar.tsx`
- Button "Unklassifizierte analysieren" → POST /api/documents/classify-batch mit filter="unclassified"
- Zeigt Anzahl unklassifizierter Dokumente als Badge

### T4: Frontend API
**File**: `frontend/src/lib/api.ts`
- `classifyDocument(id: number)` → POST /api/documents/{id}/classify
- `classifyBatch(ids?: number[])` → POST /api/documents/classify-batch

## Verification
- [ ] Einzelnes Dokument kann per Button KI-analysiert werden
- [ ] Nach Analyse: Titel, Typ, Tags, Summary erscheinen in DocumentDetail
- [ ] Batch: alle unklassifizierten Dokumente werden in Queue gestellt
- [ ] Bei nicht erreichbarem Ollama: verständliche Fehlermeldung
- [ ] Progress wird im JobProgress-Widget angezeigt
