# Plan 08-04: Re-Scan fehlgeschlagener Dokumente

## Goal
Dokumente mit Status "error" oder leerer OCR können erneut gescannt werden — einzeln oder in Batch.

## Tasks

### T1: Backend — Re-Scan Endpoint
**File**: `backend/app/documents/router.py`
- `POST /api/documents/{id}/rescan` → setzt processing_status="pending", queued neuen Parse-Job
- `POST /api/documents/rescan-errors` → alle error-Dokumente erneut scannen

**File**: `backend/app/jobs/models.py`
- Neuer `JobType.RESCAN` (oder CRAWL mit single-file payload)

**File**: `backend/app/jobs/worker.py`
- RESCAN Job-Handler: einzelnes Dokument neu parsen (mit aktuellem OCR-Code)

### T2: Frontend — Re-Scan Buttons
**File**: `frontend/src/components/DocumentDetail.tsx`
- Bei Status "error": Button "Erneut scannen"
- Ruft POST /api/documents/{id}/rescan auf

**File**: `frontend/src/components/Toolbar.tsx`
- Button "Fehler erneut scannen" (Batch)

## Verification
- [ ] Einzelnes Error-Dokument kann erneut gescannt werden
- [ ] Batch-Rescan verarbeitet alle Error-Dokumente
- [ ] Nach erfolgreichem Rescan ändert sich der Status
