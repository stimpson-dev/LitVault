    **useKeyboardShortcuts hook**:
    - Registers global keyboard event listeners (useEffect with keydown)
    - Shortcuts:
      - Ctrl+S: save current document (preventDefault to avoid browser save dialog)
      - Ctrl+Shift+S: save & close (save then navigate back)
      - Escape: close detail, navigate back to list
      - Ctrl+Left: previous document in list
      - Ctrl+Right: next document in list
    - Takes a map of {key: handler} entries
    - Cleans up listeners on unmount
    - Only active when DocumentDetailPage is mounted

    **DetailActions** (button group in detail page header):
    - "Save" button (primary)
    - "Save & Next" button (saves, then navigates to next doc in the list)
      - Needs access to the current document list order — pass via route state or context
      - If no next doc: show toast "No more documents"
    - "Save & Close" button (saves, navigates back to list)
    - "Discard" button (outline, re-fetches from API to reset form)
    - Separator
    - "More like this" button (Sparkles icon):
      - Navigates to document list with moreLike:docId filter
      - Uses the search endpoint with full_text of current doc as query
    - "Open file" button (opens file in system viewer)
    - "Classify" / "Rescan" / "Exclude" action dropdown

    **Navigation state**: Store the current document list (IDs in order) in a lightweight
    context or route state so Save & Next knows which doc comes next.
