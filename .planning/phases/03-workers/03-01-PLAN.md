  <name>Task 1: Job models and JobStore</name>
  <files>backend/app/jobs/__init__.py, backend/app/jobs/models.py</files>
  <action>
  Create `backend/app/jobs/__init__.py` (empty).

  Create `backend/app/jobs/models.py` with:

  1. `class JobType(str, Enum)` — values: "crawl", "classify"

  2. `class JobStatus(str, Enum)` — values: "queued", "processing", "done", "error"

  3. `@dataclass` `Job`:
     - id: str (uuid4 hex)
     - type: JobType
     - status: JobStatus = JobStatus.QUEUED
     - payload: dict = field(default_factory=dict)  # e.g. {"folder": "C:/docs"}
     - progress_current: int = 0
     - progress_total: int = 0
     - progress_message: str = ""
     - result: dict | None = None  # final result summary
     - error: str | None = None
     - created_at: str = field(default_factory=lambda: datetime.now(timezone.utc).isoformat())
     - started_at: str | None = None
     - finished_at: str | None = None

  4. `class JobStore`:
     - `_jobs: dict[str, Job]` — simple dict, no lock needed (single-threaded asyncio)
     - `create_job(job_type: JobType, payload: dict) -> Job` — create Job with uuid4, store, return it
     - `get_job(job_id: str) -> Job | None`
     - `list_jobs(status: JobStatus | None = None) -> list[Job]` — optionally filter by status
     - `update_progress(job_id: str, current: int, total: int, message: str = "")` — update progress fields
     - `complete_job(job_id: str, result: dict)` — set status=done, result, finished_at
     - `fail_job(job_id: str, error: str)` — set status=error, error, finished_at

  Use `from enum import Enum` and `from dataclasses import dataclass, field`.
  Use `from datetime import datetime, timezone` and `import uuid`.
  </action>

<task type="auto">
  <name>Task 2: Worker coroutine with progress callback</name>
  <files>backend/app/jobs/worker.py, backend/app/ingest/service.py</files>
  <action>
  First, update `backend/app/ingest/service.py` to support a progress callback:

  1. Add parameter to `ingest_folder`:
     ```python
     async def ingest_folder(
         self,
         folder: str,
         on_progress: Callable[[int, int, str], None] | None = None,
     ) -> IngestResult:
     ```

  2. Inside the for-loop over new_files, after processing each file, call:
     ```python
     if on_progress:
         on_progress(idx + 1, total_found, f"Processed: {meta['file_path']}")
     ```
     (where idx is the enumerate index)

  This is backwards-compatible — existing callers without on_progress still work.

  Then create `backend/app/jobs/worker.py` with:

  1. `async def process_job(job: Job, store: JobStore, settings: Settings) -> None`:
     - Set job status to PROCESSING, started_at
     - Create a fresh AsyncSession from `async_session_factory` (import from app.database)
     - try/finally to ensure session is closed
     - Match on job.type:
       - JobType.CRAWL:
         - Create IngestService(session, settings)
         - Define progress callback that calls store.update_progress(job.id, current, total, message)
         - Call ingest_folder(job.payload["folder"], on_progress=callback)
         - store.complete_job with result dict from IngestResult
       - Default: store.fail_job("Unknown job type")
     - On exception: store.fail_job(job.id, str(e)), log error

  2. `async def worker_loop(queue: asyncio.Queue, store: JobStore, settings: Settings) -> None`:
     - Loop forever: `job = await queue.get()`
     - Call process_job(job, store, settings)
     - `queue.task_done()`
     - Wrapped in try/except for CancelledError (clean exit) and Exception (log, continue)

  Use `logger = logging.getLogger("litvault.worker")`
  Import `from app.database import async_session_factory` — NOTE: you may need to check the actual export name in database.py. If it's called `async_session` or `AsyncSessionLocal`, use that name.
  </action>
  <verify>
  `cd backend && uv run python -c "from app.jobs.worker import process_job, worker_loop; print('OK')"`
  </verify>
  <done>
  Worker coroutine pulls from asyncio.Queue, creates own DB session, calls IngestService with progress callback. Handles errors gracefully. IngestService.ingest_folder now accepts optional on_progress callback.
  </done>
