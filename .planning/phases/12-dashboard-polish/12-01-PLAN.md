    **SavedViewWidget** (card per saved search):
    - Header: saved search name + document count badge + "Open" link (→ /view/:id)
    - Drag handle (GripVertical icon) on left of header
    - Body: compact list of top 5 documents from that saved search
      - Each doc: title (truncated), doc_type badge, date
      - Click navigates to /documents/:docId
    - Loading state: skeleton rows
    - Empty state: "No documents match this view"

    **Drag-and-drop** (DndContext + SortableContext from @dnd-kit/sortable):
    - Each SavedViewWidget wrapped in useSortable
    - On drag end: reorder the widgets array
    - Persist widget order to settings (new field: dashboard_widget_order: number[])
    - Drag preview: slightly rotated, reduced opacity card
    - Only the drag handle (grip icon) initiates drag

    **Data loading**:
    - Fetch all saved searches on mount
    - For each saved search: fetch top 5 results using search API with saved filters
    - Use Promise.all for parallel fetching
    - Cache results (don't re-fetch on every widget reorder)
